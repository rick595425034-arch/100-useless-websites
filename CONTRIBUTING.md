# 新增与维护约定

这个仓库采用“一个整活，一个独立项目”的方式维护。不要让不同作品共享隐式依赖，也不要为了新增一个作品改坏已经可以录制的旧作品。

## 新项目清单

- 使用连续三位编号与英文短名：`projects/003-example-name`。
- 提供 `README.md`，至少写清创意、运行命令、测试命令和安全说明。
- 提供独立的 `package.json` 与锁文件。
- 在 `catalog.json` 中增加一项，并同步根目录 README 的项目表格与进度。
- 首页不要提前写出最终包袱；短视频中的关键中文应优先保证可读性。
- 至少加入一个自动测试，保护核心表演流程与安全边界。

## 提交前检查

在仓库根目录执行：

```bash
npm test
```

再进入本次修改的项目执行：

```bash
npm install
npm test
```

## 不应提交的内容

- `node_modules`、缓存、测试输出和构建目录
- DMG、应用打包产物与本机编译的 `.node` 文件
- 录屏、临时截图和用于调试的桌面快照
- `.env`、令牌、证书、私钥或任何个人数据

桌面项目需要原生能力时，应提交可审查的源代码与构建脚本，让使用者在本机生成二进制文件。

## 兼容性

已有的独立仓库和线上部署可以继续保留。总仓库从现在开始作为系列项目的主要目录；同步旧独立仓库时，应在提交说明中注明对应项目编号。
